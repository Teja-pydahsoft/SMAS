# SMAS — Split Platform Deployment

Deploy SMAS across four free-tier services:

| Service | Platform | URL pattern |
|---------|----------|-------------|
| Frontend | [Vercel](https://vercel.com) | `https://your-app.vercel.app` |
| Backend | [Render](https://render.com) | `https://smas-backend.onrender.com` |
| AI Server | [Fly.io](https://fly.io) | `https://smas-ai-server.fly.dev` |
| Database | [MongoDB Atlas](https://www.mongodb.com/atlas) | connection string |

```
Browser → Vercel (Next.js)
              │  /api/* rewrite
              ▼
          Render (Express) ──→ Fly.io (FastAPI + InsightFace)
              │
              ▼
          MongoDB Atlas
```

---

## Step 1 — MongoDB Atlas (database)

1. Create a free account at [mongodb.com/atlas](https://www.mongodb.com/atlas).
2. Create a **M0 Free** cluster (pick a region close to you, e.g. Mumbai).
3. **Database Access** → add a user with read/write password.
4. **Network Access** → allow `0.0.0.0/0` (required for Render/Fly cloud IPs).
5. **Connect** → Drivers → copy the connection string:
   ```
   mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/smas?retryWrites=true&w=majority
   ```
   Replace `<password>` and keep database name `smas`.

## Step 2 — AI server (pick one — no Fly.io card required)

Fly.io often requires a debit card. **Use one of these instead** — full guide: **[deploy/AI-HOSTING.md](deploy/AI-HOSTING.md)**

### Recommended: Hugging Face Spaces (free, no card, 16 GB RAM)

1. [huggingface.co](https://huggingface.co) → **New Space** → **Docker**
2. Connect GitHub repo `Teja-pydahsoft/SMAS`
3. Settings → Dockerfile path: `ai-server/Dockerfile`, App port: `8000`
4. Test: `https://YOUR-SPACE.hf.space/health`

### Alternative: Render Docker (included in `render.yaml`)

Deploys as `smas-ai-server` with the backend blueprint. Free tier has only 512 MB — may be too small for InsightFace.

## Step 3 — Render (backend)

1. Push this repo to GitHub.
2. Go to [render.com](https://render.com) → **New** → **Blueprint**.
3. Connect the repo — Render reads `render.yaml` at the repo root.
4. After deploy, open the **smas-backend** service → **Environment** and set:

| Variable | Value |
|----------|-------|
| `MONGODB_URI` | Atlas connection string from Step 1 |
| `AI_SERVER_URL` | `https://smas-ai-server.fly.dev` (your Fly URL) |
| `FRONTEND_URL` | Set after Vercel deploy (Step 4), e.g. `https://smas.vercel.app` |

`JWT_SECRET` and `SUPER_ADMIN_PASSWORD` are auto-generated by the blueprint. Copy the super admin password from Render env vars for first login.

5. Wait for deploy. Test: `https://smas-backend.onrender.com/api/health`  
   Should show `"ai": "online"` once Fly is running.

**Free tier note:** Render free services sleep after ~15 min idle. First request after sleep takes ~30–60s.

---

## Step 4 — Vercel (frontend)

1. Go to [vercel.com](https://vercel.com) → **Add New Project** → import the GitHub repo.
2. Set **Root Directory** to `frontend`.
3. **Environment Variables:**

| Name | Value |
|------|-------|
| `BACKEND_URL` | `https://smas-backend.onrender.com` |

4. Deploy. Note your URL, e.g. `https://smas.vercel.app`.
5. Go back to **Render** → update `FRONTEND_URL` to your Vercel URL (comma-separate multiple origins if needed).
6. Redeploy the backend on Render (or it picks up env changes on next deploy).

Test the app at your Vercel URL. Login with the super admin credentials from Render env vars.

## Environment variable reference

### Frontend (Vercel)

| Variable | Required | Description |
|----------|----------|-------------|
| `BACKEND_URL` | Yes | Render backend URL for API rewrites |

### Backend (Render)

| Variable | Required | Description |
|----------|----------|-------------|
| `MONGODB_URI` | Yes | MongoDB Atlas connection string |
| `AI_SERVER_URL` | Yes | Fly.io AI server URL |
| `FRONTEND_URL` | Yes | Vercel URL(s), comma-separated |
| `JWT_SECRET` | Yes | Auto-generated in blueprint |
| `SUPER_ADMIN_PASSWORD` | Yes | Auto-generated in blueprint |

### AI Server (Fly.io)

Configured in `ai-server/fly.toml`. Override via `fly secrets set KEY=value` if needed.

## Known free-tier limitations

1. **Render cold starts** — backend sleeps when idle; gate scans may feel slow on first hit.
2. **Upload storage** — photos are stored on Render’s ephemeral disk and are lost on redeploy. For production persistence, add S3/Cloudinary later.
3. **Face index** — persisted on Fly volume (`/data`). Survives AI server restarts but not volume deletion.
4. **HTTPS** — Vercel and Fly provide HTTPS automatically. Camera access in the browser requires HTTPS (Vercel handles this).
