# SMAS ‚Äî Split Platform Deployment

Deploy SMAS across four free-tier services:

| Service | Platform | URL pattern |
|---------|----------|-------------|
| Frontend | [Vercel](https://vercel.com) | `https://your-app.vercel.app` |
| Backend | [Render](https://render.com) | `https://smas-backend.onrender.com` |
| AI Server | [Fly.io](https://fly.io) | `https://smas-ai-server.fly.dev` |
| Database | [MongoDB Atlas](https://www.mongodb.com/atlas) | connection string |

```
Browser ‚Üí Vercel (Next.js)
              ‚îÇ  /api/* rewrite
              ‚ñº
          Render (Express) ‚îÄ‚îÄ‚Üí Fly.io (FastAPI + InsightFace)
              ‚îÇ

```

## Step 1 ‚Äî MongoDB Atlas (database)

1. Create a free account at [mongodb.com/atlas](https://www.mongodb.com/atlas).
2. Create a **M0 Free** cluster (pick a region close to you, e.g. Mumbai).
3. **Database Access** ‚Üí add a user with read/write password.
4. **Network Access** ‚Üí allow `0.0.0.0/0` (required for Render/Fly cloud IPs).
5. **Connect** ‚Üí Drivers ‚Üí copy the connection string:
   ```
   mongodb+srv://<user>:<password>@cluster0.xxxxx.mongodb.net/smas?retryWrites=true&w=majority
   ```
   Replace `<password>` and keep database name `smas`.

## Step 2 ‚Äî Fly.io (AI server)

Requires [flyctl](https://fly.io/docs/hands-on/install-flyctl/) and a Fly account.

```bash
cd ai-server
fly auth login
fly launch --no-deploy
# Accept app name or set smas-ai-server; pick region bom (Mumbai) or nearest
fly volumes create smas_ai_data --region bom --size 1
fly deploy
fly status
```

Note the app URL, e.g. `https://smas-ai-server.fly.dev`.

**First deploy** downloads the InsightFace model (~100‚Äì500 MB) and can take several minutes. The health check has a 60s grace period.

**Free tier note:** `fly.toml` uses `buffalo_s` and 1 GB RAM to fit free allowances. For better accuracy on a paid plan, switch to `buffalo_l` and `DET_SIZE=640,640`.

## Step 3 ‚Äî Render (backend)

1. Push this repo to GitHub.
2. Go to [render.com](https://render.com) ‚Üí **New** ‚Üí **Blueprint**.
3. Connect the repo ‚Äî Render reads `render.yaml` at the repo root.
4. After deploy, open the **smas-backend** service ‚Üí **Environment** and set:

| Variable | Value |
|----------|-------|
| `MONGODB_URI` | Atlas connection string from Step 1 |
| `AI_SERVER_URL` | `https://smas-ai-server.fly.dev` (your Fly URL) |
| `FRONTEND_URL` | Set after Vercel deploy (Step 4), e.g. `https://smas.vercel.app` |

`JWT_SECRET` and `SUPER_ADMIN_PASSWORD` are auto-generated by the blueprint. Copy the super admin password from Render env vars for first login.

5. Wait for deploy. Test: `https://smas-backend.onrender.com/api/health`  
   Should show `"ai": "online"` once Fly is running.

**Free tier note:** Render free services sleep after ~15 min idle. First request after sleep takes ~30‚Äì60s.

---

## Step 4 ‚Äî Vercel (frontend)

1. Go to [vercel.com](https://vercel.com) ‚Üí **Add New Project** ‚Üí import the GitHub repo.
2. Set **Root Directory** to `frontend`.
3. **Environment Variables:**

| Name | Value |
|------|-------|
| `BACKEND_URL` | `https://smas-backend.onrender.com` |

4. Deploy. Note your URL, e.g. `https://smas.vercel.app`.
5. Go back to **Render** ‚Üí update `FRONTEND_URL` to your Vercel URL (comma-separate multiple origins if needed).
6. Redeploy the backend on Render (or it picks up env changes on next deploy).

Test the app at your Vercel URL. Login with the super admin credentials from Render env vars.

## Environment variable reference

### Frontend (Vercel)

| Variable | Required | Description |
|----------|----------|-------------|
| `BACKEND_URL` | Yes | Render backend URL for API rewrites |

### Backend (Render)

| Variable | Required | Description |
|----------|----------|-------------|
| `MONGODB_URI` | Yes | MongoDB Atlas connection string |
| `AI_SERVER_URL` | Yes | Fly.io AI server URL |
| `FRONTEND_URL` | Yes | Vercel URL(s), comma-separated |
| `JWT_SECRET` | Yes | Auto-generated in blueprint |
| `SUPER_ADMIN_PASSWORD` | Yes | Auto-generated in blueprint |

### AI Server (Fly.io)

Configured in `ai-server/fly.toml`. Override via `fly secrets set KEY=value` if needed.

## Known free-tier limitations

1. **Render cold starts** ‚Äî backend sleeps when idle; gate scans may feel slow on first hit.
2. **Upload storage** ‚Äî photos are stored on Render‚Äôs ephemeral disk and are lost on redeploy. For production persistence, add S3/Cloudinary later.
3. **Face index** ‚Äî persisted on Fly volume (`/data`). Survives AI server restarts but not volume deletion.
4. **HTTPS** ‚Äî Vercel and Fly provide HTTPS automatically. Camera access in the browser requires HTTPS (Vercel handles this).
